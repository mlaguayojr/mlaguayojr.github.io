The Problem

Every semester at Southern Connecticut State University, I'd sit down with my printed schedule and manually punch every class into Google Calendar. Course name, time, days of the week, location, professor — times seven or eight courses. Then again next semester. And the one after that.

It's not a hard task. It's just a tedious one. And the more semesters I went through, the more convinced I became that a computer should be doing this for me.

The Solution

SCSU uses a system called BannerWeb to manage student records — grades, registration, transcripts, and crucially, class schedules. It's a web portal, which means the data I needed was sitting behind a login form and a few HTML tables.

I built a Python script that logs into BannerWeb with your credentials, scrapes your registered courses for a given semester, and exports everything as an .ics file — the standard calendar format that Google Calendar, Apple Calendar, and Outlook all import natively. One command, one file, done.

The project is archived on GitHub: mlaguayojr/myscsucalendar.

How It Works

Authentication

The script uses the requests library to maintain a session across HTTP calls. Logging in is a POST to BannerWeb's login endpoint with your student username and PIN. The response is checked for a known redirect string to confirm the login succeeded before proceeding. Credentials are collected at runtime via getpass so nothing is ever hardcoded.

Parsing the Schedule

Once authenticated, the script fetches the course schedule page and hands the HTML off to BeautifulSoup. BannerWeb renders schedules as nested HTML tables, so the parser walks those tables to extract everything useful: course title, CRN, professor name and email, meeting days and times, building and room, credit hours, and the semester's start and end dates.

Days of the week are stored in BannerWeb as single-character codes — M (Monday), T (Tuesday), W (Wednesday), R (Thursday), F (Friday) — which the script maps to their iCalendar equivalents (MO, TU, WE, TH, FR) for the recurrence rules.

Generating the .ics File

Each course becomes a recurring calendar event using the icalendar library. The script calculates the correct first occurrence of each class by finding the first matching weekday on or after the semester start date, then sets a weekly recurrence rule through the semester end date. The event description includes the CRN, professor details, and credit hours.

The finished calendar is written to a .ics file named after the semester (e.g., Spring 2017.ics), ready to import into Google Calendar.

Usage

The script requires Python 3 and four dependencies:

  • requests
  • beautifulsoup4
  • icalendar
  • getpass (part of the standard library)

A typical session looks like this:

scsu = SCSU()
scsu.user()                       # prompts for your username and PIN
scsu.login()
scsu.listSemesters()              # prints the 5 most recent semesters and their codes
scsu.downloadSemester("201740")   # exports Spring 2017 as an .ics file
scsu.logout()

Import the resulting file into Google Calendar and you're done. Took about a minute instead of twenty.
