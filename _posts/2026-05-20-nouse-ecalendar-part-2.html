Recap

I was gifted an Anyuse eCalendar — a 16-inch Android touchscreen meant to serve as a household command center. Nearly every feature depends on WiFi, and the device drops its connection daily without recovering on its own. In Part 1, I tried various workarounds using automation apps, but hit a wall with root access limitations.

Diagnosis

I pulled the device logs and had Claude analyze them. The diagnosis: the WiFi scanning process is getting stuck. The WiFi chip (AIC8800) has a documented reputation for instability across platforms. The driver enters a deadlocked state and eventually the device forces a disconnection.

Fixes to Try

  1. Update WiFi Driver
  2. Factory Reset WiFi Settings
  3. Check for System Updates
  4. Disable WiFi Power Saving
  5. Reset Network Settings

Outcome

While toggling WiFi state programmatically requires root, there's another angle worth exploring: simulated taps. An automation flow could tap through the Settings UI to toggle WiFi manually — no root required. See you in Part 3.
